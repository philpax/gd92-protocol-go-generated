# GD92 Protocol for Go

A Go library implementing the GD-92/1003A/2.2 communications protocol for UK Fire Service Mobilising Systems.

## Background

This repository was created in response to [a Hacker News comment by ErroneousBosh](https://news.ycombinator.com/item?id=47472375), who described their experience building a GD92 protocol parser:

> [...] listen on TCP for GD92 packets, send handshaking to the remote endpoint, then parse the packets into Go structs delivered via channels, and encode Go structs back into packets for transmission.

This is a proof-of-concept implementation generated entirely by [Claude Code](https://claude.ai/code) from the GD-92 specification PDF—a 140-page document specifying the communications infrastructure for fire service mobilising systems, published by the UK Home Office. The spec was [retrieved from Public Contracts Scotland](https://www.publiccontractsscotland.gov.uk/NoticeDownload/DownloadDocument.aspx?id=OCT398050&idx=9), converted to text with `pdftotext`, and read directly by Claude Code. The user did not manually write any GD92 protocol code.

The goal was to demonstrate that LLMs can implement obscure, niche protocols directly from their specifications, even when zero public implementations, test vectors, or packet captures exist anywhere on the internet. All implementations of GD92 are proprietary, held by commercial fire service equipment vendors. Every test vector in this library was hand-constructed from the spec's bit layout definitions.

You can read the full Claude Code transcripts for the [planning session](https://gisthost.github.io/?f878f4f3c6d3416bbd28a32e5f4f0240/index.html) and the [execution session](https://gisthost.github.io/?528a0337ba213a93d633e501cad80743/index.html).

**This is not production software.** It has not been tested against real fire service equipment. If you're actually building fire service communications systems, you should probably not be getting your protocol libraries from GitHub repositories generated by AI.

## What is GD92?

GD-92 defines a store-and-forward messaging system for UK fire brigades. Messages—mobilisation commands, resource status updates, incident notifications, alerter controls—are wrapped in envelopes containing addressing (brigade/node/port), sequencing, priority, and BCC integrity checks. Envelopes are transferred between nodes via Message Transfer Agents (MTAs) using a simple SOH/EOT framed protocol with master/slave acknowledgement characters.

## Quick Start

### Creating and encoding an envelope

```go
msg := &gd92.MobiliseCommand{
    OpPeripherals: 0x0003, // Station sounders + lights
    ManAckReq:     true,
}
msgData, _ := msg.MarshalGD92()

env := &gd92.Envelope{
    Source:       gd92.CommsAddress{Brigade: 1, Node: 0, Port: 1},
    Destinations: []gd92.CommsAddress{{Brigade: 1, Node: 10, Port: 4}},
    Priority:     1,
    ProtVers:     1,
    AckReq:       true,
    Seq:          1,
    MessageType:  gd92.MsgMobiliseCommand,
    Message:      msgData,
}

wireData, _ := env.MarshalEnvelope()
frame := gd92.WrapFrame(wireData)
```

### Parsing a received envelope

```go
envelope := gd92.UnwrapFrame(frame)
env, _ := gd92.UnmarshalEnvelope(envelope)
msg, _ := gd92.ParseEnvelopeMessage(env)

switch m := msg.(type) {
case *gd92.MobiliseCommand:
    fmt.Printf("Mobilise! Peripherals: 0x%04x\n", m.OpPeripherals)
case *gd92.ResourceStatus:
    for _, r := range m.Resources {
        fmt.Printf("%s: status %d\n", r.Callsign, r.StatusCode)
    }
case *gd92.ACK:
    fmt.Println("ACK received")
}
```

### Running a server

```go
cfg := mta.DefaultConfig()
cfg.AckType = mta.AckSlave

srv, _ := mta.Listen(":4092", cfg)
ctx := context.Background()
go srv.Serve(ctx)

for rx := range srv.Received() {
    if rx.Err != nil {
        log.Println("error:", rx.Err)
        continue
    }
    fmt.Printf("From %s: type %d\n",
        rx.Envelope.Source, rx.Envelope.MessageType)

    // Send ACK reply
    ackEnv := &gd92.Envelope{
        Source:       rx.Envelope.Destinations[0],
        Destinations: []gd92.CommsAddress{rx.Envelope.Source},
        Seq:          rx.Envelope.Seq,
        MessageType:  gd92.MsgACK,
    }
    rx.Conn.SendEnvelope(ctx, ackEnv)
}
```

### Connecting as a client

```go
cfg := mta.DefaultConfig()
cfg.AckType = mta.AckMaster

cl, _ := mta.Dial(ctx, "firestation:4092", cfg)
defer cl.Close()

cl.Send(ctx, env)

for rx := range cl.Received() {
    // Handle responses
}
```

## Architecture

```
gd92/                         Core protocol package
  types.go                    Encoder/Decoder for word8/16/32, bool, string, compressed_string
  address.go                  CommsAddress: 3-byte packed brigade/node/port
  compression.go              ESC-based run-length compression (spec A.3)
  envelope.go                 Envelope marshal/unmarshal with BCC
  frame.go                    SOH/EOT framing constants
  message.go                  Message interface, type constants, registry
  messages.go                 All 40+ message type structs
  reason.go                   Reason code constants (Appendix D)

gd92/mta/                     Message Transfer Agent layer
  config.go                   MTA configuration (timeouts, retries, ack_type)
  conn.go                     Frame-level connection with demux read loop
  server.go                   TCP server with channel-based receive API
  client.go                   TCP client with channel-based send/receive API
```
